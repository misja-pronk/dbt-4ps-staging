# dbt_4ps_staging_package

dbt package with 73 generated staging models for [4PS Construct](https://www.4ps.nl/) (Business Central) data exported with [bc2adls](https://github.com/Bertverbeek4PS/bc2adls) to Databricks. Every model is a [streaming table](https://docs.databricks.com/aws/en/dlt/streaming-tables) that incrementally reads the CSV deltas for one source table, with typed columns, snake_case names, column documentation, and a primary-key uniqueness test.

The models are generated by [`../dbt_4ps_staging_generator`](../dbt_4ps_staging_generator/) — regenerate instead of editing them by hand if your bc2adls table set differs.

## Configuration

Point the package at your environment via vars (override in `dbt_project.yml` or `--vars`):

```yaml
vars:
  source:        # Unity Catalog volume holding the bc2adls export
    database: my_catalog
    schema: bc2adls
    volume: bc2adls_deltas
  destination:   # where the streaming tables are created
    database: my_catalog
    schema: staging
```

The bundled `profiles.yml` reads the connection from environment variables:

```bash
export DBT_DATABRICKS_HOST=<workspace>.azuredatabricks.net
export DBT_DATABRICKS_HTTP_PATH=/sql/1.0/warehouses/<warehouse-id>
export DBT_DATABRICKS_CATALOG=<catalog>
export DBT_DATABRICKS_TOKEN=<token>
```

## Run

```bash
uv sync
uv run dbt deps --profiles-dir .
uv run dbt run --profiles-dir .
uv run dbt test --profiles-dir .
```

Streaming tables require a SQL warehouse or serverless compute, and `dbt run` triggers an incremental refresh — new deltas are ingested, already-processed files are skipped.
