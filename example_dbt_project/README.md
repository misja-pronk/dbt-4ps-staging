# example_dbt_project

A **minimal, working example** of a dbt project that wires up 4PS Construct staging models generated by [`dbt-4ps-generator`](../dbt_4ps_staging_generator/). It ships three sample models (`payment_terms`, `currency`, `customer`) generated from the example CDM export in [`../_cdm`](../_cdm/).

> [!IMPORTANT]
> This is a starting point, not a shippable package. **Every 4PS Construct
> environment exports a different set of tables**, so there is no universal
> model set — you generate your own. Copy this project, regenerate the models
> for your tables (see below), and commit the result to your own repo.

Each model is a [streaming table](https://docs.databricks.com/aws/en/dlt/streaming-tables) that incrementally reads the CSV deltas for one source table, with typed columns, snake_case names, column documentation, and a primary-key uniqueness test.

## Generate models for your own tables

Point the [generator](../dbt_4ps_staging_generator/) at your bc2adls CDM manifest and write into this project's `models/staging/4ps/` directory (this replaces the sample models):

```bash
uvx dbt-4ps-generator generate \
  --manifest /path/to/your/deltas.manifest.cdm.json \
  --output-directory models/staging/4ps
```

See the [generator docs](https://misja-pronk.github.io/dbt-4ps-staging/generator/) for how to download the CDM metadata from your Unity Catalog volume.

## Configuration

Point the project at your environment via vars (override in `dbt_project.yml` or `--vars`):

```yaml
vars:
  source:        # Unity Catalog volume holding the bc2adls export
    database: my_catalog
    schema: bc2adls
    volume: bc2adls_deltas
  destination:   # where the streaming tables are created
    database: my_catalog
    schema: staging
```

The bundled `profiles.yml` reads the connection from environment variables:

```bash
export DBT_DATABRICKS_HOST=<workspace>.azuredatabricks.net
export DBT_DATABRICKS_HTTP_PATH=/sql/1.0/warehouses/<warehouse-id>
export DBT_DATABRICKS_CATALOG=<catalog>
export DBT_DATABRICKS_TOKEN=<token>
```

## Run

```bash
uv sync
uv run dbt deps --profiles-dir .
uv run dbt run --profiles-dir .
uv run dbt test --profiles-dir .
```

Streaming tables require a SQL warehouse or serverless compute, and `dbt run` triggers an incremental refresh — new deltas are ingested, already-processed files are skipped.
